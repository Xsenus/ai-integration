## 4.2. Используемые технологии

**Хранилище данных**

SQL-хранилище: PostgreSQL.

**Векторный индекс**

pgvector в составе PostgreSQL (встроенное расширение, хранение векторных полей в тех же БД; при недоступности расширения используется fallback на BYTEA на уровне схемы прототипа).

**Backend / frontend**

Backend: Python 3 / FastAPI (асинхронный REST API), SQLAlchemy + asyncpg; интеграции с внешними сервисами через HTTP‑клиенты.

Frontend: React (web-кабинет).

**Очереди / планировщик задач**

В прототипе — in‑process фоновые задачи на asyncio (create_task) и периодический цикл синхронизации (настраиваемый интервал) без внешней очереди.

**Интеграции**

CRM: Bitrix24 (импорт лидов, запись рейтингов/рекомендаций/статусов).

API‑обогащение по ИНН: DaData (провайдер реестровых данных).

LLM API (извлечение фактов, генерация материалов): внешний AI‑сервис (chat model gpt‑4o); embeddings API: text‑embedding‑3‑large (или заданная модель через ENV).

**Безопасность / аудит**

Роли и права доступа, журнал действий, логирование шагов пайплайна, лимиты по квотам.

## 4.3. Публикации/патенты/ПО Заявителя

**Патент/заявка:** заявка на патент (компьютерно-реализуемый способ/система), готовая для подачи.

**Номер/дата/наименование:** заявка в подготовке; номер и дата будут присвоены при подаче.

**ПО для ЭВМ:** регистрация ПО не оформлена; планируется регистрация программного комплекса (интеграционный контур + AI‑анализ).

**Иные результаты ИС:** база данных текстовых снапшотов сайтов и эмбеддингов, наборы отраслевых справочников/шаблонов, проектная/техническая документация.
